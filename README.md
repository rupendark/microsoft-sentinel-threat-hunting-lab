# Microsoft Sentinel Threat Hunting & Alert Triage Lab

## Overview

This project demonstrates hands-on experience with Microsoft Sentinel, Kusto Query Language (KQL), and SOC alert triage workflows. The investigation focused on analyzing security events, querying log data, validating alerts, and identifying indicators of compromise (IOCs) through threat hunting techniques.

## Objectives

* Analyze security events using Microsoft Sentinel.
* Investigate and triage security alerts.
* Perform threat hunting using KQL queries.
* Identify suspicious authentication and system activities.
* Map findings to SOC investigation workflows and MITRE ATT&CK techniques.

## Technologies Used

* Microsoft Sentinel
* Kusto Query Language (KQL)
* Azure Log Analytics Workspace
* Microsoft Security Operations Concepts
* MITRE ATT&CK Framework

## Skills Demonstrated

* Security Monitoring
* Alert Triage
* Threat Hunting
* Incident Investigation
* Log Analysis
* IOC Identification
* KQL Query Development
* Security Event Correlation
* SOC Operations

## Key Activities

* Investigated security alerts generated within Microsoft Sentinel.
* Used KQL to search, filter, and analyze security logs.
* Correlated events to identify suspicious user and system activity.
* Assessed alert severity and prioritized investigation efforts.
* Documented findings and supported incident response workflows.

## Conclusion

This project strengthened my practical understanding of Microsoft Sentinel as a cloud-native SIEM and SOAR platform. Through hands-on alert investigation and threat hunting exercises, I gained experience in log analysis, incident triage, KQL querying, and SOC analyst workflows commonly used in enterprise security operations.
